cbk - 命令行备份任务管理工具

功能：用于管理备份任务，包括添加、运行、编辑、删除备份任务，查看任务日志，显示任务详情等。

用法：
  cbk [参数] <子命令> [参数]

子命令：
  list                列出所有备份任务
  run                 运行指定备份任务
  add                 添加新的备份任务
  delete              删除指定备份任务
  edit                编辑指定备份任务
  log                 查看备份任务的日志
  show                显示指定备份任务的详细信息
  version             显示当前版本信息

参数：
  -h, --help          显示帮助信息

子命令参数：

  list:
    无参数，列出所有备份任务的基本信息。

  run:
    -id, --id <int>   指定要运行的备份任务ID（必填）。

  add:
    -n, --name <string>       指定备份任务的名称（必填）。
    -t, --target <string>     指定备份的目标目录（必填）。
    -b, --backup <string>     指定备份文件存放的父目录路径（可选）。
    -k, --keep <int>          指定保留的备份数量（默认为3）。

  delete:
    -id, --id <int>   指定要删除的备份任务ID（必填）。
    -n, --name <string>       指定要删除的备份任务名称（可选，若同时指定ID和名称，则按ID优先删除）。

  edit:
    -id, --id <int>   指定要编辑的备份任务ID（必填）。
    -n, --name <string>       指定新的备份任务名称（可选）。
    -k, --keep <int>          指定新的保留备份数量（默认为3，若不指定则保持原设置）。

  log:
    -l, --limit <int> 指定显示的日志行数（默认为10）。

  show:
    -id, --id <int>   指定要显示详细信息的备份任务ID（必填）。

  unpack:
    -id, --id <int>   指定要解压的备份文件ID（必填）。
    -o, --o <string>   指定解压后的目录路径（可选，若不指定则解压到当前目录下）。
    -v, --v <string>   指定解压的版本ID（通过 show 命令查看）。

  version:
    无参数，显示当前版本信息。

  help:
    无参数，显示帮助信息。

示例：
  cbk list
    列出所有备份任务的基本信息。

  cbk run -id 1
    运行ID为1的备份任务。

  cbk add -n "每日备份" -t "/data" -b "/backup" -k 7
    添加一个名为“每日备份”的备份任务，目标目录为“/data”，备份文件存放在“/backup”，保留最近7个备份。

  cbk delete -id 2
    删除ID为2的备份任务。

  cbk edit -id 3 -n "周备份" -k 4
    将ID为3的备份任务名称改为“周备份”，并将保留数量设置为4。

  cbk log -l 20
    查看最近20行备份任务的日志。

  cbk show -id 4
    显示ID为4的备份任务的详细信息。

  cbk unpack -id 5 -o "/data/unpacked" -v 151nmgd1

  cbk version
    显示当前版本信息。

注意：
- 在使用子命令时，请确保参数的正确性，避免因参数错误导致任务执行失败。
- 对于某些操作（如删除和编辑），建议在执行前仔细确认任务ID或名称，以避免误操作。
